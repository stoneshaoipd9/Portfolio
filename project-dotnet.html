Auto Part Warehouse Inventory System


(.NET Project: Wei Shao, Yas Ansari)

Description

An Auto Part Warehouse Inventory System will have different access levels for different users. The administrator is the only user who can grant/modify other users’ access levels. After successful login, the user can access 4 different options such as Sale, Purchase, Inventory and Reports.
On choosing the Sales/Purchase option, the user will have Sale/Purchase window open. After all required information is entered and the SUBMIT button is click, the new changes will be implemented in the inventory table. The sold stock will be the ones to be shipped out.
When user chooses Inventory option, he will see the large view of the inventory that is read only and he cannot modify the information. At any point of time, the user can go to Report window and print the desired information.

List of technologies

  • WPF, C#
  • SQL Database via Azure
  • Multiple windows
  • Ribbon instead menus
  • TabControl
